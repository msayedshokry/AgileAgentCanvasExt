Rationale per surface
---------------------

1. `.corpus-3d-spinner` — border-top-color via `--vscode-focusBorder, #007FD8`
   HARDCODED fallback. The 6px rotating accent ring is a non-text UI
   component; the upstream token clears 3:1 in Dark+/Light+/HC-Dark.

2. `.corpus-3d-error-wrap` + `.corpus-3d-error-detail` — color via
   `--vscode-errorForeground` (theme-aware red; no HARDCODED because
   the existing Autonomy.css chip pattern already uses this token).

3. `.corpus-3d-search-box` + `.corpus-3d-search-input` + `.corpus-3d-search-icon`
   + `.corpus-3d-search-clear` + `.corpus-3d-match-count` — 3 chrome tiles
   designed as translucent overlays on Dark+/HC-Dark
   (`rgba(0, 0, 0, 0.6)`). Light+ override flips them to opaque
   `var(--vscode-editor-background, #FFFFFF)` because semi-transparent
   black composites to ~#808080 on white editor bg — sub-3:1 UI-floor.
   The override restores visual hierarchy (chip stays distinct from
   parent card bg) AND lifts the fg-vs-bg contrast to ~16:1.

4. `.corpus-3d-search-clear` color (Round-2 Universal-form fix) —
   `var(--vscode-descriptionForeground, #505050)` Universal form with
   HARDCODED `#505050` fallback. TOKS[--vscode-descriptionForeground][Dark+]
   = `#B4B4B4` ~7.07:1 vs `#1E1E1E`; [Light+] = `#717171` ~4.46:1 vs
   `#FFFFFF` (clears 3:1 with margin); [HC-Dark] = `#FFFFFF` ~17:1 vs
   `#000000`. The HARDCODED fallback fires if a partial theme omits the
   upstream token (e.g. exotic Tokyonight variants).

5. `.corpus-3d-phase-swatch--{0..3}` 4 variants — HARDCODED per-phase
   colors mirror the `PHASE_COLORS` const in Corpus3DView.tsx L24.
   TOKENIZATION INTENTIONALLY REJECTED for the BASE form: `PHASE_COLORS`
   is also consumed by the THREE rendered NodeMesh material at L141
   (`new THREE.Color(...)` writes can't read CSS vars). Tokenizing the
   swatch would silently desync the chip visual from the 3D shape color.
   The const is the single source-of-truth; both files must stay in
   sync. Light+ override rebinds phases 1/2/3 (cyan/orange/green) to
   deeper tones (#0e7490 / #b45309 / #15803D) clearing 3:1 vs
   `#FFFFFF` editor-bg; phase-0 (purple `#ab47bc`) retained as-is
   (~4.31:1 already clears 3:1 UI-floor in Light+).

Co-location rationale (Kanban.css not Corpus3DView.css)
------------------------------------------------------

`.corpus-3d-*` lives in Kanban.css not Corpus3DView.css because the
3D-force-graph overlay UI renders inside AgenticKanbanApp.tsx (the
chrome palette is co-located with the kanban view, not the corpus
view). Kanban.css already owns the AgenticKanban chrome palette
(approval-banner, kanban-card chrome, status dots). Splitting the
`.corpus-3d-*` block into a separate Corpus3DView.css would require a
new dedicated stylesheet import chain without functional benefit.
