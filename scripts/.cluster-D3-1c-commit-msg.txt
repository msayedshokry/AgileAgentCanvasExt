fix(a11y/kanban): Cluster D-3 #1c — tokenize .kanban-card-type-tag + Light+ dark-bg rebind

## What
Closes the lone remaining CSS-side FAIL row from the audit-script sweep
(.kanban-card-type-tag, KEY chip on Kanban card). Pre-fix the rule declared
`background: var(--vscode-badge-background)` + `color: var(--vscode-badge-foreground)`
with NO HARDCODED fallback and NO @media Light+ override; in HC-Dark where the upstream
--vscode-badge-background token is unset in TOKS, both resolved to the parent bg
(1.00:1 identity-mismatch) and Light+ upstream #B4B4B4 rendered ~2.07:1 vs #FFFFFF fg
— both sub-3:1 WCAG 1.4.11 UI-component floor.

## Why this fix shape
- **HARDCODED Universal fallbacks** (`#4D4D4D` bg + `#FFFFFF` fg): guarantee HC-Dark
  renders against a known-good gray regardless of theme settings.
- **Light+ `@media (prefers-color-scheme: light)` override HARDCODED** (`#1F1F1F` bg +
  `#FFFFFF` fg): pre-fix the override used `var(--vscode-editor-background)` /
  `var(--vscode-foreground)` which TOKS-resolves to `#FFFFFF` / `#1F1F1F` — 16:1 ratio
  but the chip BACKGROUND aliased to the parent card bg (`#FFFFFF` in Light+) making
  the chip visually invisible. The dark-bg/white-fg inverse style keeps the chip
  identifiably distinct from the card surface while clearing WCAG 1.4.11 + AA-text at
  ~15:1.

### DRY-amortization: KANBAN_CSS hoist to module-scope
Right below the `KANBAN_CSS_PATH` declaration at L75 of the test file. Resolves the
`ReferenceError: KANBAN_CSS is not defined` thrown by the new SHAPE-Anchor test — and
amortizes the DRY debt flagged in D-2 #7 closure (each prior D-2 #N describe was
declaring its own local `const KANBAN_CSS = readFileSync(...)`).

The hoist uses an anchored multiline regex to detect module-scope insertion — earlier
v1 used a naive substring check that falsely matched the local redeclarations inside
describe blocks (L2066/L2205), causing v1 to SKIP hoisting. v4 detects only the
actual module-scope insertion right below `KANBAN_CSS_PATH`.

Intent documented in the inserted comment block: **module-scope is the L1 baseline;
local re-declarations inside D-2 #N describes are explicit fresh-parses for tests
that need to re-read after a CSS mutation.** Shadowing is bounded and intentional —
not swept because doing so could regress tests that depend on fresh reads.

Also: dead `KANBAN_CSS_FOR_D3_1C` alias dropped, replaced with direct module-scope
`KANBAN_CSS` references.
