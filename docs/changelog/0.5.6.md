# Changelog Detail — 0.5.6 (Unreleased)

> Companion detail file for the **0.5.6** entries in [`CHANGELOG.md`](../CHANGELOG.md). The top-level `CHANGELOG.md` carries only headline bullets under **Unreleased** so the section stays under the 1,500-word cap enforced by `npm run lint:changelog`; this document holds the full user-facing mechanics for each highlight.

## Feature: Terminal reconnect toast with persisted-output fallback (#35)

When a terminal-based agent run survives a VS Code restart, the canvas now shows a reconnected toast the moment the underlying terminal comes back online and auto-opens the Terminal modal pre-populated with the most recent output for the matching artifact. Until now, the extension silently re-attached the listener on the backend but the user never saw any signal that the run was live again — every recovered session looked like a ghost.

- A success toast appears as soon as a previously orphaned session is matched and re-attached, with the Terminal modal auto-opened when no modal is already visible and there is recovered output to show.
- The recovered output is sourced from the in-memory accumulator while the session is still alive, then falls back transparently to a per-session buffer persisted to disk (capped at 200 KB) when the recovery happens after a VS Code restart.
- The Terminal modal populates its lines with the recovered output so you immediately see what the agent wrote before the crash or restart, instead of staring at an empty terminal.
- The reconnector is hardened against false positives: when the live-output hook is unavailable on the current VS Code build, the session is correctly marked interrupted instead of silently reporting success.
- 7 new unit tests in the recovery suite cover the success, failure, missing-reconnector, no-pid, and replacement-terminal paths.

## Fixed: Dev-story confirm modal dropped; agent prompts reference artifact file path

Dragging a story from Ready for Dev to In-Progress no longer opens a Run/Skip confirmation modal — the drag already commits intent, so the extra dialog was redundant friction. As a bonus, the agent prompt that the dev-story workflow hands the model no longer embeds the full artifact JSON; it references the on-disk file path and tells the agent to use the standard file-read tool when it actually needs the content.

- The Ready-for-Dev → In-Progress transition now ships with the confirmation prompt disabled — the dropped modal matches the contract the rest of the Agentic Kanban already relied on.
- Agent prompts now surface artifact ID, title, type, status, parent epic link (when present), and the on-disk file path with an explicit hint to use the read tool — small, consistent across story, epic, requirement, use case, PRD, architecture, product brief, test case, and test strategy.
- Full artifact JSON is no longer embedded in the prompt, keeping the token footprint predictable on large epics where the embedded JSON used to dominate the prompt budget.
- A graceful fallback emits a minimal summary if the store cannot resolve a file path, so downstream workflows still get the bare essentials and never see an empty prompt block.

## Feature: Agent Sessions sidebar panel with ACP session manager

A new Agent Sessions section in the Activity Bar gives a single live view of every running agent — multi-agent team sessions, monitored single-agent runs, terminal-driven workflows, and Kanban drops from the Agentic Kanban. Each row shows its source, current phase, status pill, and last-update time so you can see what is actually running and spot stuck sessions at a glance without flipping between views.

- New sidebar entry sits alongside the existing Artifacts and Workflow Progress trees, registered through the standard Activity Bar mechanism so the same view menus and visibility toggles apply.
- Rows are unified across ACP team sessions, health-monitored single-agent runs, terminal-driven workflows, and Kanban drops so you see one consolidated list, not four parallel ones that drift out of sync.
- A live snapshot is pushed on activation and incremental updates are pushed on every state change — the webview never polls, so refresh cost stays flat regardless of session count.
- A 30-second heartbeat keeps the panel fresh for sessions whose backends do not emit explicit state changes, without flooding the webview with redundant snapshots.
- A new ACP session manager and team orchestrator back the panel with full TypeScript type definitions and an accompanying vitest suite covering the public surface.
- A Cucumber regression locks the dev-story lane-transition skip-confirmation contract so the panel can keep relying on the drag-to-launch invariant.

## Feature: Ponytail extends to terminal, antigravity, and goal-decomposer

The minimalist hierarchy now reaches every remaining AI agent prompt path. Chat conversation, workflow execution, and ACP multi-agent handoffs already carried the full heuristic via prior commits; this round finishes the job for the headless terminal path, antigravity's autonomous + default execution modes, and adds a deliberately narrow carve-out for the JSON-shape goal-decomposition prompt.

- **Terminal executor (`buildTerminalPrompt`)** — Headless CLI agents running BMAD workflows end-to-end (Claude Code, Codex, Gemini CLI, Aider, OpenCode) now open their prompts with the full heuristic block plus an "apply before adding code" callout, co-located with the verdict contract. Skill content and partial-artifact fallbacks also receive the heuristic so it always reaches the agent regardless of which prompt-branch is taken. A 13-assertion vitest regression guard locks the verbatim constant, all six hierarchy items, all five NOT-lazy-about boundaries, the verification rule, and both fallback paths.

- **Antigravity orchestrator (`buildGuideContent`)** — Autonomous and default execution modes now carry the full heuristic. A `ponytailSection` guard (`mode !== 'interactive' ? PONYTAIL_SECTION : ''`) ensures interactive mode stays untouched — it is a guided dialog, not a coding session, and the prompt there is already short and conversational. The guard is set just after `const mode = resolveExecutionMode(executionHints);` so all three return templates (autonomous, default, interactive) render consistently; only autonomous + default inject the heuristic, interactive renders an empty string. A 23-assertion vitest suite (added via `src/antigravity/antigravity-orchestrator.test.ts`) locks the conditional gate, autonomous injection, default injection, interactive exclusion, and all four `resolveExecutionMode` hint combinations.

- **Goal decomposer (carve-out, narrow)** — Instead of injecting the full 400-token `PONYTAIL_HEURISTICS` constant into the goal-decomposition prompt — which would dilute a narrow JSON-shape prompt (the LM would try to apply ALL the hierarchy rules, not the JSON shape itself) — `wireGoalDecomposerHooks().decompose` now carries a single 12-token sentence: "If the goal fits in a single story, return one." This is the soft form of necessity-first from the Ponytail heuristic: it nudges the LM toward a single story when the goal is itself single-story, without imposing the higher-level abstraction-sizing rules that would add noise to a JSON-constrained extraction prompt. The carve-out sits between the existing "2-5 concrete, actionable user stories" sentence and the "Return ONLY a JSON array..." sentence, so single-story decisions are made before the array-shape implication kicks in.

- **Final coverage matrix** — Six of seven prompt sites now carry some form of the hierarchy: Chat (full), workflow execution (full), ACP multi-agent (full), headless terminal agents (full), antigravity autonomous + default (full), and goal decomposition (carve-out). The remaining gap — antigravity interactive mode — is intentional: guided dialogs do not write code, and the prompt there is already brief and conversational.

- **No regressions** — All three layered test suites pass: 13 terminal-executor assertions in `terminal-executor.test.ts`, 23 antigravity-orchestrator assertions in `antigravity-orchestrator.test.ts`, and the 13 ponytail-heuristics unit tests in `ponytail-heuristics.test.ts`. `vitest.config.ts` adds `'src/antigravity/**/*.test.ts'` to the include list so the new test file is discovered alongside the existing workflow-tier guards. Typecheck clean. All 810+ lane-transition Cucumber scenarios unaffected.
